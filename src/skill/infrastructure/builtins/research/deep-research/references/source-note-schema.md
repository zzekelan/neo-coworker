# Source Note Schema

Store source notes with these exact fields and order:

1. ID
2. Type
3. Title
4. URI/Path
5. Retrieved at
6. Reliability
7. Related findings
8. Excerpt
9. Notes

Use `web` for public pages, `docs` for reference documentation, and `files` for local workspace or user-provided files.

ID prefixes match the type: `W001` for web, `D001` for docs, and `F001` for files.

Keep `Excerpt` short. Summarize or quote only the smallest useful passage, and never copy large raw materials.

Use `Notes` for access limits, freshness concerns, private-file cautions, or follow-up checks.
